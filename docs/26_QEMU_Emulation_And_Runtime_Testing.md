## 1. Overview & Architecture

QEMU (Quick EMUlator) is the cornerstone of Buildroot's **offline validation** story. Rather
than flashing every build to physical hardware, developers emulate complete target machines
in software, running the exact root filesystem produced by Buildroot — kernel, initramfs,
and all userspace — inside a virtual machine on the build host.

Buildroot integrates QEMU at two levels:

| Level | Mechanism | Purpose |
|---|---|---|
| **User-space emulation** | `qemu-<arch>-static` | Run individual target binaries on host |
| **System emulation** | `qemu-system-<arch>` | Boot full kernel + rootfs as VM |

This chapter focuses on **system emulation**: booting a complete Buildroot image inside
QEMU, verifying it is functional, and automating that verification in CI pipelines.

### Why EXT2 for QEMU?

`BR2_TARGET_ROOTFS_EXT2=y` produces a raw disk image that QEMU can present as a
virtio-blk or IDE block device. The kernel mounts it as the root filesystem exactly as it
would on real hardware. EXT2 (and its journalled sibling EXT4) are universally supported
by the Linux kernel without initramfs tricks, making them the lowest-friction choice for
emulation targets.

## 2. Buildroot Configuration for QEMU

### 2.1 Minimal `defconfig` Skeleton

```
# Target architecture
BR2_x86_64=y                          # or BR2_aarch64=y / BR2_riscv=y
BR2_x86_x86_64=y

# Toolchain
BR2_TOOLCHAIN_BUILDROOT_GLIBC=y
BR2_TOOLCHAIN_BUILDROOT_CXX=y         # needed for C++ programs

# Kernel — use a QEMU-friendly config
BR2_LINUX_KERNEL=y
BR2_LINUX_KERNEL_DEFCONFIG="x86_64"   # or "virt" for arm64 / "rv64" for RISC-V
BR2_LINUX_KERNEL_NEEDS_HOST_OPENSSL=y

# Root filesystem
BR2_TARGET_ROOTFS_EXT2=y
BR2_TARGET_ROOTFS_EXT2_4=y            # use EXT4 variant
BR2_TARGET_ROOTFS_EXT2_SIZE="256M"

# QEMU host tools (so `output/host/bin/qemu-system-*` is available)
BR2_PACKAGE_HOST_QEMU=y
BR2_PACKAGE_HOST_QEMU_SYSTEM_MODE=y
BR2_PACKAGE_HOST_QEMU_ARCH_ENABLE_X86_64=y   # adjust per target

# Optional: virtio drivers built into the kernel
# (set in the kernel config fragment)
```

### 2.2 Kernel Config Fragment for QEMU (`board/qemu/x86_64/linux.config`)

```
CONFIG_VIRTIO=y
CONFIG_VIRTIO_PCI=y
CONFIG_VIRTIO_BLK=y
CONFIG_VIRTIO_NET=y
CONFIG_VIRTIO_CONSOLE=y
CONFIG_9P_FS=y
CONFIG_9P_FS_POSIX_ACL=y
CONFIG_NET_9P=y
CONFIG_NET_9P_VIRTIO=y
CONFIG_EXT4_FS=y
CONFIG_TTY=y
CONFIG_SERIAL_8250=y
CONFIG_SERIAL_8250_CONSOLE=y
```

### 2.3 Running `make` for a QEMU Target

```bash
# Configure
make qemu_x86_64_defconfig     # Buildroot ships a ready-made defconfig

# Build everything (kernel + rootfs)
make -j$(nproc)

# Artifacts produced:
#   output/images/bzImage        (x86_64 kernel)
#   output/images/rootfs.ext4    (EXT4 disk image)
#   output/images/start-qemu.sh  (launch script, auto-generated by post-image.sh)
```

## 3. Filesystem Images: EXT2/EXT4 & Beyond

### 3.1 How Buildroot Builds the Image

```
┌─────────────────────────────────────────────────────────────────┐
│  make  →  fakeroot  →  makedevs  →  mkfs.ext4  →  rootfs.ext4   │
└─────────────────────────────────────────────────────────────────┘
```

The key Makefile target lives in `fs/ext2/ext2.mk`:

```makefile
# Simplified from Buildroot source
EXT2_SIZE  = $(call qstrip,$(BR2_TARGET_ROOTFS_EXT2_SIZE))
EXT2_OPTS  = -d $(TARGET_DIR) -r 1 -N 0 -m 5 \
             -L "rootfs" -O ^64bit

$(BINARIES_DIR)/rootfs.ext4: $(TARGET_DIR) $(HOST_DIR)/sbin/mkfs.ext4
	$(HOST_DIR)/sbin/mkfs.ext4 -q \
	    $(EXT2_OPTS) \
	    $(BINARIES_DIR)/rootfs.ext4 \
	    "$(EXT2_SIZE)"
```

### 3.2 Inspecting and Modifying the Image on the Host

```bash
# Mount for inspection (no root needed with guestmount)
guestmount -a output/images/rootfs.ext4 -i --ro /mnt/rootfs

# Resize if needed
truncate -s +128M output/images/rootfs.ext4
resize2fs output/images/rootfs.ext4

# Check filesystem integrity
e2fsck -f output/images/rootfs.ext4
```

### 3.3 Alternative Image Formats

| Format | Buildroot Option | QEMU Flag | Notes |
|---|---|---|---|
| EXT2/4 (raw) | `BR2_TARGET_ROOTFS_EXT2=y` | `-drive file=rootfs.ext4,format=raw` | Simplest |
| CPIO initramfs | `BR2_TARGET_ROOTFS_CPIO=y` | `-initrd rootfs.cpio` | No block device needed |
| SquashFS | `BR2_TARGET_ROOTFS_SQUASHFS=y` | `-drive …` + overlay | Read-only, smaller |
| Tar | `BR2_TARGET_ROOTFS_TAR=y` | Manual unpack | CI artefact distribution |

## 4. `start-qemu.sh` — Launch Script Deep Dive

Buildroot's board support packages ship a `start-qemu.sh` (generated by `post-image.sh`)
that wires up QEMU with the correct flags for each architecture. Understanding this script
is essential for customising emulation or plugging it into CI.

### 4.1 Canonical `start-qemu.sh` for `x86_64`

```bash
#!/usr/bin/env bash
# board/pc/start-qemu.sh  (annotated)

BINARIES_DIR="${0%/*}/"
cd "${BINARIES_DIR}"

# ── Machine & CPU ───────────────────────────────────────────────────────────
QEMU_ARCH=x86_64
QEMU_MACHINE=pc          # full ISA PC

# ── Memory ──────────────────────────────────────────────────────────────────
RAM=512                  # MiB — keep low for CI runners

# ── Kernel + rootfs ─────────────────────────────────────────────────────────
KERNEL=bzImage
ROOTFS=rootfs.ext4

# ── Network (user-mode, no tap needed) ──────────────────────────────────────
NETDEV="-netdev user,id=net0,hostfwd=tcp::2222-:22 \
        -device virtio-net-pci,netdev=net0"

# ── Serial console → stdout/stdin ───────────────────────────────────────────
CONSOLE="-serial stdio -append \"root=/dev/vda rw console=ttyS0\""

exec qemu-system-${QEMU_ARCH}       \
    -M ${QEMU_MACHINE}              \
    -m ${RAM}                       \
    -kernel ${KERNEL}               \
    -drive file=${ROOTFS},if=virtio,format=raw \
    -append "root=/dev/vda rw console=ttyS0 quiet" \
    ${NETDEV}                       \
    -nographic                      \
    -no-reboot
```

### 4.2 `start-qemu.sh` for `aarch64`

```bash
#!/usr/bin/env bash
BINARIES_DIR="${0%/*}/"
cd "${BINARIES_DIR}"

exec qemu-system-aarch64            \
    -M virt                         \
    -cpu cortex-a57                 \
    -m 512                          \
    -kernel Image                   \
    -drive file=rootfs.ext4,if=none,format=raw,id=hd0 \
    -device virtio-blk-pci,drive=hd0 \
    -append "root=/dev/vda rw console=ttyAMA0" \
    -netdev user,id=net0            \
    -device virtio-net-pci,netdev=net0 \
    -nographic                      \
    -no-reboot
```

### 4.3 `start-qemu.sh` for `RISC-V 64`

```bash
#!/usr/bin/env bash
BINARIES_DIR="${0%/*}/"
cd "${BINARIES_DIR}"

exec qemu-system-riscv64            \
    -M virt                         \
    -m 512                          \
    -bios fw_jump.elf               \
    -kernel Image                   \
    -drive file=rootfs.ext4,format=raw,id=hd0 \
    -device virtio-blk-device,drive=hd0 \
    -append "root=/dev/vda rw console=ttyS0 earlycon=sbi" \
    -netdev user,id=net0            \
    -device virtio-net-device,netdev=net0 \
    -nographic                      \
    -no-reboot
```

> **Note:** RISC-V requires OpenSBI (`fw_jump.elf`) as the BIOS/bootloader layer.
> Buildroot builds this automatically when `BR2_TARGET_OPENSBI=y` is set.

## 5. Target Architectures: x86_64, aarch64, RISC-V

### 5.1 Architecture Comparison Matrix

| Feature | x86_64 | aarch64 | RISC-V 64 |
|---|---|---|---|
| QEMU binary | `qemu-system-x86_64` | `qemu-system-aarch64` | `qemu-system-riscv64` |
| Machine type | `pc` or `q35` | `virt` | `virt` |
| Console device | `ttyS0` | `ttyAMA0` | `ttyS0` |
| Kernel image | `bzImage` | `Image` | `Image` |
| Boot firmware | BIOS (built-in) | EDK2 or built-in | OpenSBI (`fw_jump.elf`) |
| Buildroot `BR2_` | `BR2_x86_64=y` | `BR2_aarch64=y` | `BR2_riscv=y` + `BR2_RISCV_64=y` |
| Virtio transport | PCI | PCI | MMIO (`-device`) |

### 5.2 Cross-Architecture Build Commands

```bash
# x86_64
make qemu_x86_64_defconfig && make -j$(nproc)

# aarch64
make qemu_aarch64_virt_defconfig && make -j$(nproc)

# RISC-V 64
make qemu_riscv64_virt_defconfig && make -j$(nproc)
```

### 5.3 Using `-cpu` to Model Specific Cores

```bash
# Cortex-A72 (Raspberry Pi 4 approximation)
qemu-system-aarch64 -M virt -cpu cortex-a72 ...

# Emulate host CPU features (fastest, not reproducible)
qemu-system-x86_64 -M pc -cpu host -enable-kvm ...

# RISC-V with specific extensions
qemu-system-riscv64 -M virt -cpu rv64,v=true,vlen=128 ...
```

## 6. Automated Boot Testing with `expect`

`expect` is a Tcl-based scripting tool that drives interactive programs through their
stdin/stdout. It is the traditional — and still highly effective — way to automate boot
verification of embedded Linux images without modifying the target rootfs.

### 6.1 Basic Boot Verification Script

```tcl
#!/usr/bin/expect -f
# boot_test_basic.exp — Verify the system boots and reaches a login prompt

set timeout 120
set qemu_cmd {
    qemu-system-x86_64
    -M pc
    -m 256
    -kernel bzImage
    -drive file=rootfs.ext4,if=virtio,format=raw
    -append {root=/dev/vda rw console=ttyS0 quiet}
    -nographic
    -no-reboot
}

# Spawn QEMU
spawn {*}$qemu_cmd

# ── Phase 1: Kernel decompression ────────────────────────────────────────────
expect {
    "Booting the kernel"    { puts "✓ Kernel decompressed" }
    timeout                 { puts "✗ TIMEOUT: kernel never started"; exit 1 }
}

# ── Phase 2: Root filesystem mounted ─────────────────────────────────────────
expect {
    "EXT4-fs"               { puts "✓ EXT4 root mounted" }
    "Kernel panic"          { puts "✗ PANIC: rootfs not mounted"; exit 1 }
    timeout                 { puts "✗ TIMEOUT: waiting for rootfs"; exit 1 }
}

# ── Phase 3: Login prompt ─────────────────────────────────────────────────────
expect {
    "login:"                { puts "✓ Login prompt reached" }
    timeout                 { puts "✗ TIMEOUT: no login prompt"; exit 1 }
}

# ── Phase 4: Login and run a command ─────────────────────────────────────────
send "root\r"
expect {
    "#"                     { puts "✓ Logged in as root" }
    timeout                 { puts "✗ TIMEOUT: login failed"; exit 1 }
}

send "uname -a\r"
expect {
    -re {Linux \S+ \d+\.\d+} {
        puts "✓ Kernel version: $expect_out(0,string)"
    }
    timeout                 { puts "✗ TIMEOUT: uname failed"; exit 1 }
}

# ── Shutdown cleanly ──────────────────────────────────────────────────────────
send "poweroff\r"
expect eof
puts "✓ Boot test PASSED"
exit 0
```

### 6.2 Advanced Multi-Phase Test Suite

```tcl
#!/usr/bin/expect -f
# boot_test_suite.exp — Full regression suite: network, storage, process table

package require Tcl 8.5

set timeout 180
set pass_count 0
set fail_count 0
set log_file "boot_test_[clock format [clock seconds] -format %Y%m%d_%H%M%S].log"

log_file -noappend $log_file

proc check { description pattern } {
    global pass_count fail_count timeout
    expect {
        -re $pattern {
            puts "  \[PASS\] $description"
            incr pass_count
            return 1
        }
        "Kernel panic" {
            puts "  \[FAIL\] $description — Kernel panic"
            incr fail_count
            return 0
        }
        timeout {
            puts "  \[FAIL\] $description — Timeout (${timeout}s)"
            incr fail_count
            return 0
        }
    }
}

proc run_cmd { cmd expected_re } {
    global pass_count fail_count
    send "$cmd\r"
    expect {
        -re $expected_re { incr pass_count; return 1 }
        timeout          { incr fail_count; return 0 }
    }
}

# ── Spawn QEMU ────────────────────────────────────────────────────────────────
spawn qemu-system-x86_64              \
    -M pc                             \
    -m 512                            \
    -kernel bzImage                   \
    -drive file=rootfs.ext4,if=virtio,format=raw \
    -append "root=/dev/vda rw console=ttyS0"     \
    -netdev user,id=net0              \
    -device virtio-net-pci,netdev=net0 \
    -nographic -no-reboot

puts "\n=== Boot Test Suite ===\n"

# Phase 1: Boot sequence
puts "-- Phase 1: Boot Sequence --"
check "Kernel starts"        {Booting the kernel}
check "Root FS mounted"      {EXT4-fs.*mounted}
check "Init started"         {(systemd|Syslogd|Starting|Welcome)}
check "Login prompt"         {login:}

# Phase 2: Login
puts "\n-- Phase 2: Authentication --"
send "root\r"
check "Root shell"           {\#\s*$}

# Phase 3: System information
puts "\n-- Phase 3: System Information --"
run_cmd "uname -r"   {\d+\.\d+\.\d+}
run_cmd "cat /proc/cpuinfo | grep 'model name' | head -1" {model name}
run_cmd "free -m"    {Mem:}
run_cmd "df -h /"    {/dev/vda}
run_cmd "ip link"    {eth0|lo}

# Phase 4: Functional checks
puts "\n-- Phase 4: Functional Checks --"
run_cmd "ls /bin /sbin /usr/bin" {(ls|cat|sh)}
run_cmd "cat /proc/mounts"       {/dev/root}
run_cmd "dmesg | grep -c error"  {\d+}

# Phase 5: Shutdown
puts "\n-- Phase 5: Shutdown --"
send "poweroff\r"
set timeout 30
expect {
    eof { puts "  \[PASS\] Clean shutdown" ; incr pass_count }
    timeout { puts "  \[WARN\] Shutdown timeout" }
}

# Summary
puts "\n=== Results: $pass_count passed, $fail_count failed ==="
puts "=== Log written to: $log_file ===\n"
exit [expr {$fail_count > 0 ? 1 : 0}]
```

## 7. C Example: Embedded Runtime Health Check

This program runs inside the QEMU guest. It checks CPU features, memory, and
filesystem availability, printing a structured report — ideal as an `expect`-triggered
post-boot validation binary.

### `healthcheck.c`

```c
/*
 * healthcheck.c — Embedded runtime health checker for QEMU guest
 *
 * Compile (cross): ${CROSS_COMPILE}gcc -O2 -static -o healthcheck healthcheck.c
 * Usage: ./healthcheck [--json]
 */

#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <errno.h>
#include <sys/utsname.h>
#include <sys/statvfs.h>
#include <sys/sysinfo.h>
#include <sys/types.h>
#include <fcntl.h>
#include <unistd.h>

#define PASS  "[PASS]"
#define FAIL  "[FAIL]"
#define WARN  "[WARN]"

/* ── Helpers ──────────────────────────────────────────────────────────────── */

static int check_file_exists(const char *path)
{
    return access(path, F_OK) == 0;
}

static long read_proc_value(const char *path, const char *key)
{
    FILE *f = fopen(path, "r");
    if (!f) return -1;

    char line[256];
    long value = -1;
    while (fgets(line, sizeof(line), f)) {
        if (strncmp(line, key, strlen(key)) == 0) {
            sscanf(line + strlen(key), "%ld", &value);
            break;
        }
    }
    fclose(f);
    return value;
}

/* ── Check functions ──────────────────────────────────────────────────────── */

static int check_kernel(void)
{
    struct utsname u;
    if (uname(&u) < 0) {
        fprintf(stderr, "%s uname() failed: %s\n", FAIL, strerror(errno));
        return 0;
    }
    printf("%s Kernel: %s %s (%s)\n", PASS, u.sysname, u.release, u.machine);
    return 1;
}

static int check_memory(unsigned long min_mb)
{
    struct sysinfo si;
    if (sysinfo(&si) < 0) {
        fprintf(stderr, "%s sysinfo() failed: %s\n", FAIL, strerror(errno));
        return 0;
    }

    unsigned long total_mb = (si.totalram * si.mem_unit) / (1024 * 1024);
    unsigned long free_mb  = (si.freeram  * si.mem_unit) / (1024 * 1024);

    if (total_mb < min_mb) {
        printf("%s Memory: %lu MB total (< required %lu MB)\n",
               FAIL, total_mb, min_mb);
        return 0;
    }
    printf("%s Memory: %lu MB total, %lu MB free\n", PASS, total_mb, free_mb);
    return 1;
}

static int check_rootfs(const char *mount_point, unsigned long min_mb_free)
{
    struct statvfs st;
    if (statvfs(mount_point, &st) < 0) {
        fprintf(stderr, "%s statvfs(%s): %s\n", FAIL, mount_point, strerror(errno));
        return 0;
    }

    unsigned long free_mb = (st.f_bavail * st.f_frsize) / (1024 * 1024);
    unsigned long total_mb = (st.f_blocks * st.f_frsize) / (1024 * 1024);

    if (free_mb < min_mb_free) {
        printf("%s RootFS (%s): %lu MB free / %lu MB total — too low\n",
               FAIL, mount_point, free_mb, total_mb);
        return 0;
    }
    printf("%s RootFS (%s): %lu MB free / %lu MB total\n",
           PASS, mount_point, free_mb, total_mb);
    return 1;
}

static int check_virtio_devices(void)
{
    const char *virtio_paths[] = {
        "/sys/bus/virtio/drivers/virtio_blk",
        "/sys/bus/virtio/drivers/virtio_net",
        NULL
    };
    int all_ok = 1;
    for (int i = 0; virtio_paths[i]; i++) {
        if (check_file_exists(virtio_paths[i])) {
            printf("%s VirtIO: %s\n", PASS, virtio_paths[i]);
        } else {
            printf("%s VirtIO: %s (missing)\n", WARN, virtio_paths[i]);
            all_ok = 0;
        }
    }
    return all_ok;
}

static int check_cpu_count(void)
{
    long n = sysconf(_SC_NPROCESSORS_ONLN);
    if (n < 1) {
        printf("%s CPU: could not determine count\n", FAIL);
        return 0;
    }
    printf("%s CPU: %ld processor(s) online\n", PASS, n);
    return 1;
}

/* ── Main ─────────────────────────────────────────────────────────────────── */

int main(int argc, char *argv[])
{
    int json_mode = (argc > 1 && strcmp(argv[1], "--json") == 0);
    int total = 0, passed = 0;

    if (!json_mode) {
        printf("=== QEMU Guest Health Check ===\n\n");
    }

    /* Run checks */
    struct { const char *name; int result; } checks[] = {
        { "kernel",  check_kernel()              },
        { "memory",  check_memory(64)            },
        { "rootfs",  check_rootfs("/", 8)        },
        { "virtio",  check_virtio_devices()      },
        { "cpu",     check_cpu_count()           },
    };

    total = sizeof(checks) / sizeof(checks[0]);
    for (int i = 0; i < total; i++)
        if (checks[i].result) passed++;

    if (json_mode) {
        printf("{\"total\":%d,\"passed\":%d,\"failed\":%d}\n",
               total, passed, total - passed);
    } else {
        printf("\n=== %d/%d checks passed ===\n", passed, total);
    }

    return (passed == total) ? EXIT_SUCCESS : EXIT_FAILURE;
}
```

### Integration with `expect`

```tcl

## 8. C++ Example: QEMU Device Probe & Reporter

This C++ program enumerates PCI and platform devices visible inside the QEMU guest
by reading `sysfs`, then produces a structured summary. It demonstrates using the STL
in a cross-compiled Buildroot target binary.

### `device_probe.cpp`

```cpp
/*
 * device_probe.cpp — Enumerate QEMU guest devices via sysfs
 *
 * Compile: ${CROSS_COMPILE}g++ -std=c++17 -O2 -static -o device_probe device_probe.cpp
 */

#include <iostream>
#include <fstream>
#include <sstream>
#include <vector>
#include <string>
#include <filesystem>
#include <optional>
#include <iomanip>
#include <cstring>
#include <cerrno>

namespace fs = std::filesystem;

// ── Data structures ──────────────────────────────────────────────────────────

struct PciDevice {
    std::string bdf;       // Bus:Device.Function  e.g. "0000:00:01.0"
    std::string vendor_id;
    std::string device_id;
    std::string class_id;
    std::string driver;

    std::string describe() const {
        return bdf + "  vendor=" + vendor_id
                   + " device=" + device_id
                   + " class=" + class_id
                   + (driver.empty() ? "" : " drv=" + driver);
    }
};

struct VirtioDevice {
    std::string name;      // e.g. "virtio0"
    std::string device_type;
    std::string driver;
};

// ── Helpers ──────────────────────────────────────────────────────────────────

static std::string read_sysfs(const fs::path& p)
{
    std::ifstream f(p);
    if (!f) return {};
    std::string s;
    std::getline(f, s);
    // Strip leading 0x if present
    if (s.size() > 2 && s[0] == '0' && s[1] == 'x')
        s = s.substr(2);
    return s;
}

static std::string resolve_symlink_name(const fs::path& p)
{
    std::error_code ec;
    auto target = fs::read_symlink(p, ec);
    if (ec) return {};
    return target.filename().string();
}

// ── PCI enumeration ──────────────────────────────────────────────────────────

static std::vector<PciDevice> enumerate_pci()
{
    std::vector<PciDevice> devices;
    const fs::path pci_root = "/sys/bus/pci/devices";

    std::error_code ec;
    if (!fs::exists(pci_root, ec)) return devices;

    for (const auto& entry : fs::directory_iterator(pci_root, ec)) {
        if (ec) break;

        PciDevice dev;
        dev.bdf       = entry.path().filename().string();
        dev.vendor_id = read_sysfs(entry.path() / "vendor");
        dev.device_id = read_sysfs(entry.path() / "device");
        dev.class_id  = read_sysfs(entry.path() / "class");
        dev.driver    = resolve_symlink_name(entry.path() / "driver");
        devices.push_back(std::move(dev));
    }
    return devices;
}

// ── VirtIO enumeration ───────────────────────────────────────────────────────

static std::vector<VirtioDevice> enumerate_virtio()
{
    std::vector<VirtioDevice> devices;
    const fs::path virtio_root = "/sys/bus/virtio/devices";

    std::error_code ec;
    if (!fs::exists(virtio_root, ec)) return devices;

    for (const auto& entry : fs::directory_iterator(virtio_root, ec)) {
        if (ec) break;

        VirtioDevice dev;
        dev.name        = entry.path().filename().string();
        dev.device_type = read_sysfs(entry.path() / "device");
        dev.driver      = resolve_symlink_name(entry.path() / "driver");
        devices.push_back(std::move(dev));
    }
    return devices;
}

// ── ASCII table renderer ─────────────────────────────────────────────────────

static void print_table_header(const std::string& title,
                                const std::vector<std::string>& cols,
                                const std::vector<int>& widths)
{
    std::cout << "\n  " << title << "\n";

    // Top border
    std::cout << "  +";
    for (int w : widths) std::cout << std::string(w + 2, '-') << "+";
    std::cout << "\n";

    // Header row
    std::cout << "  |";
    for (size_t i = 0; i < cols.size(); i++) {
        std::cout << " " << std::left << std::setw(widths[i]) << cols[i] << " |";
    }
    std::cout << "\n";

    // Separator
    std::cout << "  +";
    for (int w : widths) std::cout << std::string(w + 2, '=') << "+";
    std::cout << "\n";
}

static void print_table_row(const std::vector<std::string>& cells,
                             const std::vector<int>& widths)
{
    std::cout << "  |";
    for (size_t i = 0; i < cells.size(); i++) {
        std::string cell = cells[i];
        if ((int)cell.size() > widths[i])
            cell = cell.substr(0, widths[i] - 1) + "…";
        std::cout << " " << std::left << std::setw(widths[i]) << cell << " |";
    }
    std::cout << "\n";
}

static void print_table_footer(const std::vector<int>& widths)
{
    std::cout << "  +";
    for (int w : widths) std::cout << std::string(w + 2, '-') << "+";
    std::cout << "\n";
}

// ── Main ─────────────────────────────────────────────────────────────────────

int main()
{
    std::cout << "╔══════════════════════════════════════════╗\n";

## 9. Rust Example: Boot Test Watchdog Daemon

This Rust program runs as a lightweight watchdog daemon inside the QEMU guest. It
periodically signals liveness to the QEMU virtual watchdog (`/dev/watchdog`), and
separately communicates boot completion status over a Unix socket so the `expect`
script can confirm the system is stable.

### `Cargo.toml`

```toml
[package]
name    = "boot_watchdog"
version = "0.1.0"
edition = "2021"

[[bin]]
name = "boot_watchdog"

[profile.release]
opt-level = "z"      # Minimise binary size for embedded target
lto       = true
panic     = "abort"

# No std dependencies — keep the binary small and static-link friendly
[dependencies]
# (none — pure std only)
```

### `src/main.rs`

```rust
//! boot_watchdog — QEMU guest watchdog + boot-status IPC daemon
//!
//! Build: cargo build --release --target aarch64-unknown-linux-musl
//!
//! Responsibilities:
//!   1. Open /dev/watchdog and keep it fed (pet it every FEED_INTERVAL_S)
//!   2. Run a Unix-domain socket server on SOCK_PATH
//!   3. Respond to "STATUS\n" queries with JSON boot metrics
//!   4. Shut down cleanly on SIGTERM / SIGINT

use std::{
    fs::{File, OpenOptions},
    io::{self, Read, Write},
    net::Shutdown,
    os::unix::net::{UnixListener, UnixStream},
    path::Path,
    sync::{
        atomic::{AtomicBool, AtomicU64, Ordering},
        Arc,
    },
    thread,
    time::{Duration, Instant, SystemTime, UNIX_EPOCH},
};

// ── Constants ─────────────────────────────────────────────────────────────────

const WATCHDOG_DEV:      &str = "/dev/watchdog";
const SOCK_PATH:         &str = "/run/boot_watchdog.sock";
const FEED_INTERVAL_S:    u64 = 10;   // pet the watchdog every 10 s
const WATCHDOG_TIMEOUT_S: u64 = 30;   // must be > FEED_INTERVAL_S

// ── Shared state ──────────────────────────────────────────────────────────────

struct State {
    boot_epoch:   u64,
    feed_count:   AtomicU64,
    running:      AtomicBool,
}

impl State {
    fn new() -> Arc<Self> {
        Arc::new(Self {
            boot_epoch: SystemTime::now()
                            .duration_since(UNIX_EPOCH)
                            .unwrap_or_default()
                            .as_secs(),
            feed_count:  AtomicU64::new(0),
            running:     AtomicBool::new(true),
        })
    }
}

// ── Watchdog feeder thread ────────────────────────────────────────────────────

fn watchdog_thread(state: Arc<State>) {
    // Try to open the kernel watchdog device.
    // If absent (common in QEMU without -watchdog flag), log and idle.
    let mut wdog: Option<File> = match OpenOptions::new()
        .write(true)
        .open(WATCHDOG_DEV)
    {
        Ok(f) => {
            eprintln!("[watchdog] Opened {WATCHDOG_DEV}, timeout {}s", WATCHDOG_TIMEOUT_S);
            Some(f)
        }
        Err(e) => {
            eprintln!("[watchdog] {WATCHDOG_DEV} unavailable: {e} — running in dry mode");
            None
        }
    };

    let interval = Duration::from_secs(FEED_INTERVAL_S);

    while state.running.load(Ordering::Relaxed) {
        thread::sleep(interval);

        if let Some(ref mut f) = wdog {
            // Writing any byte to /dev/watchdog resets the countdown timer
            match f.write_all(b"\x01") {
                Ok(_) => {
                    let n = state.feed_count.fetch_add(1, Ordering::Relaxed) + 1;
                    eprintln!("[watchdog] Pet #{n}");
                }
                Err(e) => {
                    eprintln!("[watchdog] Feed error: {e}");
                }
            }
        } else {
            state.feed_count.fetch_add(1, Ordering::Relaxed);
        }
    }

    // Graceful stop: write 'V' magic to disable the watchdog before closing
    if let Some(ref mut f) = wdog {
        let _ = f.write_all(b"V");
        eprintln!("[watchdog] Watchdog disabled (magic 'V' written)");
    }
}

// ── Status socket server ──────────────────────────────────────────────────────

fn handle_client(mut stream: UnixStream, state: &Arc<State>) {
    let mut buf = [0u8; 64];
    if stream.read(&mut buf).is_err() {
        return;
    }

    let query = std::str::from_utf8(&buf)
        .unwrap_or("")
        .trim_matches('\0')
        .trim();

    if query == "STATUS" {
        let uptime = SystemTime::now()
            .duration_since(UNIX_EPOCH)
            .unwrap_or_default()
            .as_secs()
            .saturating_sub(state.boot_epoch);

        let feeds = state.feed_count.load(Ordering::Relaxed);

        let response = format!(
            "{{\"status\":\"ok\",\"uptime_s\":{uptime},\"watchdog_feeds\":{feeds}}}\n"
        );
        let _ = stream.write_all(response.as_bytes());
    } else {
        let _ = stream.write_all(b"{\"error\":\"unknown query\"}\n");
    }

    let _ = stream.shutdown(Shutdown::Both);
}

fn socket_server(state: Arc<State>) -> io::Result<()> {
    // Remove stale socket from a previous run
    if Path::new(SOCK_PATH).exists() {
        std::fs::remove_file(SOCK_PATH)?;
    }

    let listener = UnixListener::bind(SOCK_PATH)?;
    eprintln!("[socket] Listening on {SOCK_PATH}");

    // Set non-blocking so we can check `running` flag
    listener.set_nonblocking(true)?;

    while state.running.load(Ordering::Relaxed) {
        match listener.accept() {
            Ok((stream, _)) => {
                let s = Arc::clone(&state);
                thread::spawn(move || handle_client(stream, &s));
            }
            Err(ref e) if e.kind() == io::ErrorKind::WouldBlock => {
                thread::sleep(Duration::from_millis(100));
            }
            Err(e) => {
                eprintln!("[socket] Accept error: {e}");
            }
        }
    }

    let _ = std::fs::remove_file(SOCK_PATH);
    Ok(())
}

// ── Signal handling (minimal, via /proc/self) ─────────────────────────────────

fn install_ctrlc(state: Arc<State>) {
    // On embedded Linux we often can't use signal-hook crate.
    // Spawn a thread that polls /proc/self/status for simplicity,
    // or rely on the OS to kill us (no-reboot QEMU terminates on poweroff).
    //
    // For a real deployment, use signalfd(2) or sigaction(2) via libc.
    let _ = state; // placeholder — SIGTERM handled by OS shutdown
}

// ── Main ──────────────────────────────────────────────────────────────────────

fn main() {
    let state = State::new();

    eprintln!(
        "[boot_watchdog] Starting — boot epoch {}, feed interval {}s",
        state.boot_epoch, FEED_INTERVAL_S
    );

    // Watchdog feeder thread
    {
        let s = Arc::clone(&state);
        thread::spawn(move || watchdog_thread(s));
    }

    install_ctrlc(Arc::clone(&state));

    // Socket server runs on the main thread
    if let Err(e) = socket_server(Arc::clone(&state)) {
        eprintln!("[boot_watchdog] Fatal: {e}");
        std::process::exit(1);
    }

    eprintln!("[boot_watchdog] Exiting cleanly");
}
```

## 11. Troubleshooting & Common Pitfalls

### 11.1 Kernel Panics at Boot

```
Kernel panic - not syncing: VFS: Unable to mount root fs on unknown-block(0,0)
```

**Cause:** The rootfs device name in `-append` does not match the actual device.

| Driver | Device | `-append` argument |
|---|---|---|
| `virtio_blk` | `/dev/vda` | `root=/dev/vda` |
| IDE (`-drive if=ide`) | `/dev/sda` | `root=/dev/sda` |
| NVMe | `/dev/nvme0n1` | `root=/dev/nvme0n1` |

**Fix:** Ensure `CONFIG_VIRTIO_BLK=y` is in the kernel config and use `if=virtio` in QEMU.

### 11.2 `expect` Timeout Failures in CI

```tcl
# Increase global timeout for slow CI runners
set timeout 300

# Use specific timeouts per phase
expect {
    "login:"  { ... }
    timeout   { ... }   ;# This timeout overrides the global one locally
}
```

Also check that the QEMU process gets enough host RAM and CPUs:

```bash
# In CI: allocate at least 1 host core per guest vCPU
qemu-system-x86_64 -smp 2 -m 512 ...
```

### 11.3 Rust Cross-Compilation for the Target

```bash
# Install the musl target for fully static binaries (recommended for embedded)
rustup target add aarch64-unknown-linux-musl

# Point Cargo at the Buildroot cross-compiler
cat >> ~/.cargo/config.toml << 'EOF'

cargo build --release --target aarch64-unknown-linux-musl
```

### 11.4 Network Not Available Inside Guest

Verify that:
1. `CONFIG_VIRTIO_NET=y` is in the kernel config.
2. The QEMU command includes `-netdev user,id=net0 -device virtio-net-pci,netdev=net0`.
3. The guest's init system brings up `eth0` (check `/etc/network/interfaces` or `/etc/init.d/S40network`).

## 12. Summary

QEMU emulation is Buildroot's **fastest feedback loop** between a source change and a
verified running system. This chapter covered the complete workflow:

**Configuration** — `BR2_TARGET_ROOTFS_EXT2=y` (with the EXT4 variant recommended for
journalling) combined with `BR2_PACKAGE_HOST_QEMU=y` gives you both the disk image and
the emulator binary inside `output/host/`. Kernel config fragments must explicitly enable
`CONFIG_VIRTIO_BLK`, `CONFIG_VIRTIO_NET`, and the appropriate console driver.

**`start-qemu.sh`** — Each Buildroot board BSP ships a launch script that encapsulates all
QEMU flags: machine type, CPU model, memory, kernel, drive, network, and serial console.
For `x86_64` the machine is `pc` with `ttyS0`; for `aarch64` it is `virt` with
`ttyAMA0`; for RISC-V 64 it adds the OpenSBI firmware (`fw_jump.elf`) and the `earlycon`
boot parameter.

**`expect` automation** — Tcl `expect` scripts spawn QEMU as a subprocess and pattern-match
against boot output, turning subjective "did it boot?" observations into objective
pass/fail CI results. Multi-phase scripts cover kernel start → rootfs mount → init
completion → login → functional commands → clean shutdown.

**C programs** running inside the guest (like `healthcheck.c`) provide structured
machine-readable output that `expect` can parse, decoupling the test logic from raw
dmesg parsing. **C++ programs** (like `device_probe.cpp`) leverage `std::filesystem` to
enumerate `sysfs` devices and confirm that QEMU's virtualised PCI and VirtIO devices
were correctly probed by the kernel. **Rust programs** (like `boot_watchdog`) demonstrate
modern embedded-safe concurrency — a watchdog feeder thread alongside a Unix socket
server — all compiled into a zero-dependency static binary via the musl target.

Together, these pieces form a **fully automated, architecture-agnostic validation
pipeline** that can run on any CI host supporting KVM (for speed) or software
emulation (for portability), turning every Buildroot build into a verified, bootable,
functionally tested embedded Linux image.
